# 3_Deployment_Scripts 기술 명세서 (Definition of Done: DOD.md)

현재 정상적으로 연결되어 구동 중인 배포 파이프라인(Pipeline)과 에러 추적 방법을 정리한 명세서입니다.

## 1. 배포 파이프라인 개요
* **수행 업무:** 대시보드 프론트엔드 자산과 정리된 엑셀/CSV 데이터들을 Github 원격 저장소(`kbj1057-a11y/wealthfa_Dashboard`)에 동기화(Push)하여, 외부 환경에서 최신 상태로 접근 가능하도록 하는 원클릭 연결망.

## 2. 구동 중인 핵심 스크립트 자산
* **`배포하기.bat`:** 지정된 가상환경(`rpa_venv`)의 파이썬 엔진을 로드하여 자동 배포 파이썬 스크립트를 즉각 구동하는 트리거 파일.
* **`upload_to_github.bat`:** Git 초기화부터 `main` 브랜치 연동, Commit 및 최종 원격지(origin) Push까지 5단계로 세분화하여 실행하는 자동화 배치 스크립트.

## 3. 유지보수 및 에러 추적 가이드
* **경로(Path) 이탈 에러 (`지정된 경로를 찾을 수 없습니다` 발생 시):** 폴더 구조 개편에 따라 스크립트 파일이 `3_Deployment_Scripts`로 이동한 상황이다. 배치 파일 내 `cd /d "%~dp0.."`(상위 루트 폴더 기준) 위치 보정 로직이 훼손되었는지 가장 먼저 확인한다.
* **Git Push 충돌 (Reject 오류 발생 시):** 원격 저장소(Github)에 다른 내역이 먼저 올라가 충돌이 발생한 경우. `git push -f` 등의 파괴적 명령어를 남용하지 말고, 에러 발생 시 최상위 규칙(제4규칙)에 따라 Backup 폴더의 무결성을 먼저 파악한 후, 코드 리뷰(Pull & Merge) 메시지를 대표님에게 보고하는 형식으로 대응한다.
