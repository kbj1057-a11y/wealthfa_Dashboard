# 3_Deployment_Scripts 실무 가이드 (guide.md)

이 문서는 대시보드 및 RPA 결과물을 서버(웹) 또는 외부 환경으로 최종 배포할 때 에이전트가 준수해야 하는 무결성 지침입니다.

## 1. 배포 전 '핑(Ping)' 무결성 검증 (Pre-Flight Check)
* **사전 데이터 점검:** 배포 스크립트(`배포하기.bat` 등)를 실행하거나 푸시하기 전에, 대시보드의 화면을 구성하는 핵심 데이터 파일(`data1.csv`, `data2.csv` 등)과 `index.html`이 실제 제자리에 생성되어 있는지 경로 무결성을 자동 검사하라. 누락된 파일이 있다면 즉시 배포 프로세스를 중단(Fail-Fast)하고 로그를 남겨라.

## 2. 에러는 잡고 노이즈는 숨겨라 (Clean Console)
* **콘솔 정리:** Git Push 작업 등 배포 스크립 구동 시, 불필요하게 터미널 창(CMD)을 덮는 복잡한 시스템 경고(Warning)는 숨김(Suppress) 처리하고, 직관적인 진척률("1/5 진행 중...", "배포 완료!")만 명확히 출력하여 실행자의 혼란을 방지하라.

## 3. 경로(Path) 독립성 유지
* **상대 경로 절대 원칙:** 배치 파일(`.bat`) 및 배포용 파이썬 스크립트 내부에 특정 PC에 종속되는 하드코딩된 절대 경로(`C:\Users\...` 등)를 기재해서는 안 된다. 항상 실행 파일 위치 기반의 상대 경로(`%~dp0` 등)를 사용하여 어떤 환경에서도 어긋나지 않도록 설계하라.
