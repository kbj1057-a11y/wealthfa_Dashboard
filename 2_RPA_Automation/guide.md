# 2_RPA_Automation 실무 가이드 (guide.md)

이 문서는 `2_RPA_Automation` 폴더 내에서 반복 자동화 업무 및 데이터 병합 작업을 수행할 때 반드시 지켜야 하는 원본 보호(Data Integrity) 지침입니다.

## 1. 원본 데이터 보호 및 저장 원칙 (필수)
* **원본 불변 방호벽:** 사용자가 지정한 입력 대상(원본 파일)은 어떠한 경우에도 로직 내에서 수정, 덮어쓰기, 삭제 등 파괴적인 조작을 가해서는 절대 안 된다. Dataframe 조작은 오직 파이썬 메모리 공간 내에서만 수행한다.
* **출력명 컨벤션:** 모든 데이터 가공이 끝난 후 추출되는 결과 파일명은 그 파일명 끝에 예외 없이 `_통합본`, `_processed`, 혹은 날짜 태그(`{YYYYMM}`)를 붙여 **완벽히 분리된 별도 파일 상태**로 생성되어야 한다.

## 2. 데이터 위생 및 맵핑 표준화
* **인코딩 무조건 통일:** 다종다양한 환경에서 뽑힌 원본 데이터를 합친 후 결과물을 저장할 때(CSV 등의 export) 인코딩은 반드시 `utf-8-sig`를 한글 보존 기본 규격으로 채택해, 연계된 대시보드의 파서 에러를 예방한다.
* **명칭 강제 검수:** 생명/손해보험사마다 중복되거나 다른 컬럼명('수정보험료' vs '환산성적')은 병합 전 공통 딕셔너리(Mapping Table)를 통해 강제로 검수하고 1차 병합을 거쳐야 한다.
