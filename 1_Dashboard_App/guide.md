# 1_Dashboard_App 실무 가이드 (guide.md)

이 문서는 `1_Dashboard_App` 폴더 내에서 작업할 때 에이전트가 최우선으로 준수해야 하는 절대적인 기술 규칙 및 행동 강령입니다.

## 1. 데이터 로드 및 경로 절대 원칙 (필수)
* **상대 경로 고정:** 데이터를 불러올 때는 무조건 상대 경로(`./` 또는 `../2_RPA_Automation/`)를 사용하여, 로컬 환경은 물론 서버 배포 시에도 파일 누락 오류가 발생하지 않도록 한다.
* **유연한 파서 적용:** 데이터 파일의 인코딩(주로 `UTF-8`, 필요시 `EUC-KR` 등)과 구분자(탭 `\t`, 쉼표 `,`)를 자동으로 감지하고 유연하게 대응하는 파서(Parser) 로직을 기본 사양으로 채택하여 하드코딩된 에러를 방지한다.

## 2. 하위 폴더별 목적 및 구분 (라우팅 원칙)
이 대시보드 폴더는 배포 목적에 따라 두 개의 하위 조직으로 완전히 분리되어 작동한다. 작업을 지시받으면 헷갈리지 말고 아래 목적지에 맞게 진입하라.
**[긴급 질문 의무] 대시보드 관련 작업을 지시받았을 때, 대표님이 특정 폴더(1번 또는 2번)를 명시하지 않았다면, 무작정 코드를 수정하지 말고 사전에 반드시 "대표님, 1번(Streamlit)을 수정할까요, 아니면 2번(Netlify 배포용 HTML)을 수정할까요?"라고 확인(Ask first)할 것.**

### 📁 1_Streamlit_Native (파이썬 구동 본진)
* **목적:** 웰스FA 내부망이나 Streamlit Share에서 파이썬 서버 기반으로 구동하기 위한 원본 앱.
* **작업 파일:** `dashboard.py` 등 순수 Streamlit 문법과 파이썬 코드로 구성된 파일.
* **특징:** 파일과 코드가 방대하며 Python 연산(Pandas)을 풀로 사용함.

### 📁 2_Netlify_Static (서버리스/정적 배포용)
* **목적:** Netlify 같은 정적 웹 호스팅(HTML/JS만 지원)에 올리기 위해 가볍게 구워낸(Export) 버전. 파이썬 서버 없이 브라우저 단독으로 작동해야 함.
* **작업 파일:** `export_마스터피스.py` (변환기) 및 결과물 `index.html`.
* **특징:** Streamlit 의존성을 끊어내고, 오직 HTML 화면과 순수 JavaScript(Vanilla JS)만으로 데이터를 파싱하고 렌더링해야 함.

## 3. 레이아웃 및 성능 원칙
* **가독성과 속도 최우선:** 화려한 장식이나 무거운 애니메이션을 추가하는 것보다는, 데이터의 시인성(직관적인 파악)과 웹 페이지의 로딩/렌더링 속도를 최우선으로 고려하여 코드를 작성한다.
* **미니멀리즘:** 불필요한 디자인 요소(과도한 그림자 효과, 의미 없는 아이콘 등)는 철저히 배제하며, 대표님의 명시적인 시각적 효과 추가 지시가 있을 때만 해당 요소를 반영한다.
