# 1_Dashboard_App 기술 명세서 (Definition of Done: DOD.md)

이 문서는 `1_Dashboard_App` 폴더의 지식 자산화를 위한 기술 명세서입니다. 추측을 배제하고 현재 실제 구현되어 정상 작동 중인 상태만을 기록하며, 기능 업데이트 시 실시간으로 최신화해야 합니다.

## 1. 프로젝트 개요
* **목적:** 웰스FA 생명/손해보험사 수수료 분석 및 에이전트(FC)별 업적 데이터를 직관적으로 시각화하고, 누구나 쉽게 접근하여 확인할 수 있는 무설치 기반의 정적 웹 대시보드(Static HTML) 시스템을 구축 및 유지보수함.

## 2. 기술 스택 및 데이터 규격
* **사용 언어/프레임워크:** Python (데이터 가공 및 Streamlit 기반 시나리오 구성), HTML/CSS/Vanilla JavaScript (사용자 배포용 렌더링). (React, Node.js 제외)
* **데이터 규격:**
  * 파일 형식: CSV
  * 인코딩: `UTF-8` (스트림릿 덤프/파서 기준)
  * 로딩 방식: HTML 코드 내장 방식 또는 클라이언트 사이드에서 상대 경로(`./data1.csv` 등)를 통한 Fetch 로딩 구조.

## 3. 현재 구현 완료된 기능 목록
1. VVIP Gold 디자인 테마를 적용한 반응형 UI 레이아웃.
2. 수수료 및 MVP(최우수 실적자) 데이터 테이블 및 차트 렌더링.
3. Streamlit 앱을 단일 정적 `index.html` 파일로 Export 하는 스크립트 기능 (`export_마스터피스.py`).
4. 날짜 필터링 및 토글 버튼을 통한 조건별 데이터 조회 기능.
5. 보안 접속을 위한 비밀번호 입력 게이트(비밀번호: 1057) 기능.

## 4. 유지보수 및 에러 추적 가이드
* **숫자가 0으로 뜨거나 데이터가 비어 보일 때:** 가장 먼저 대시보드가 참조하는 CSV 파일(`data1.csv`, `data2.csv` 등)의 인코딩 문제(BOM 유무 등)와 구분자(쉼표 vs 탭) 문제를 확인하라. 파이썬 파서나 HTML 내 JavaScript Fetch 로직 상에서 파일 경로가 어긋났는지 점검한다.
* **페이지 레이아웃이 깨질 때:** 모바일 뷰(뷰포트)용 CSS 미디어 쿼리나 VVIP Gold 테마 스타일 블록(`update_theme.py` 등)의 최근 변경 사항을 Rollback 하거나 리뷰한다.
* **로딩이 멈출 때:** 개발자 도구(F12) 콘솔 창을 열어 네트워크 오류(404 Not Found) 여부를 확인하고, `./` 경로가 제대로 잡히는지 점검한다.
